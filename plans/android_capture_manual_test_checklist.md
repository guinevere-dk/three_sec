# Android 촬영 화면 수동 테스트 체크리스트

## 1. 테스트 목적
- [`CaptureScreen`](lib/screens/capture_screen.dart) 기준으로 촬영 기본 기능의 정상 동작 여부를 확인한다.
- 저장 경로 및 앨범 반영은 [`VideoManager.saveRecordedVideo()`](lib/managers/video_manager.dart:751) 기준으로 검증한다.

## 2. 테스트 범위
- 촬영 시작 종료
- 3초 고정 녹화
- 플래시 토글
- 줌 버튼 동작
- 탭 포커스 노출 슬라이더
- 전후면 카메라 전환
- 설정 모달 품질 비율 변경
- 촬영 후 앨범 반영
- 권한 거부 재시도 흐름

## 3. 사전 조건
- Android 실기기 1대 이상
- 앱 최초 설치 상태와 권한 거부 상태를 각각 재현 가능해야 함
- 기본 앨범 `일상` 존재 확인
- 네트워크 비활성 상태에서도 촬영 저장이 가능한지 확인 가능해야 함

## 4. 테스트 케이스

### TC-01 카메라 초기 진입
- 단계
  - 앱 실행 후 하단 탭에서 촬영 진입
- 기대 결과
  - 카메라 프리뷰가 표시됨
  - 초기 오류 시 `Retry` 버튼이 노출됨

### TC-02 촬영 시작 및 자동 종료
- 단계
  - 셔터 버튼 1회 탭
  - 카운트 UI 변화 관찰
- 기대 결과
  - 녹화 시작 직후 `REC` 또는 카운트 UI 표시
  - 약 3초 내 자동 종료
  - 중복 탭 시 비정상 종료 또는 크래시 없음

### TC-03 수동 조기 종료
- 단계
  - 녹화 시작 후 3초 이전에 셔터 재탭
- 기대 결과
  - 즉시 정지 처리
  - 저장 실패 없이 클립 생성

### TC-04 플래시 토글
- 단계
  - 플래시 버튼 반복 탭
- 기대 결과
  - 아이콘 상태가 `off` `torch`로 번갈아 변경
  - 기기 미지원 환경에서는 앱이 죽지 않고 로그만 남김

### TC-05 줌 버튼
- 단계
  - `0.5x` `1.0x` `3.0x` 순서 탭
- 기대 결과
  - 선택 스타일이 변경됨
  - 지원 범위 밖 배율은 내부 clamp 처리로 오류 없음

### TC-06 탭 포커스 노출
- 단계
  - 프리뷰 임의 지점 탭
  - 노출 슬라이더 조절
- 기대 결과
  - 포커스 링 애니메이션 표시
  - 노출 슬라이더가 일정 시간 후 자동 숨김
  - 조절 중 크래시 없음

### TC-07 카메라 전환
- 단계
  - 전환 버튼으로 전후면 반복 변경
  - 녹화 중 전환 시도
- 기대 결과
  - 녹화 중 전환 시 안전 정지 후 재초기화
  - 컨트롤러 dispose 예외가 치명 오류로 이어지지 않음

### TC-08 설정 모달 비율 변경
- 단계
  - 설정 열기 후 `9:16` `3:4` `1:1` 선택
- 기대 결과
  - 프리뷰 크롭 비율이 즉시 변경
  - 왜곡 없이 중앙 크롭 유지

### TC-09 설정 모달 화질 변경
- 단계
  - `4K` `1080` `720` 순환 선택
- 기대 결과
  - 적용 중 인디케이터 표시
  - 실패 시 이전 프리셋으로 복구
  - 화면 멈춤 없이 재초기화 완료

### TC-10 앨범 선택 및 저장 반영
- 단계
  - 앨범 드롭다운에서 대상 앨범 선택
  - 촬영 1회 수행
  - 라이브러리 탭 이동 후 해당 앨범 확인
- 기대 결과
  - 새 클립이 선택 앨범에 생성
  - 정렬은 최신순

### TC-11 권한 거부 시나리오
- 단계
  - 카메라 또는 마이크 권한 거부 후 앱 재진입
- 기대 결과
  - 초기화 실패 메시지 표시
  - `Retry` 동작 확인
  - 권한 허용 후 정상 복구
