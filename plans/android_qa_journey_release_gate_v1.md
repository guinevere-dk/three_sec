# 3s 앱 Android QA 체크리스트 + 실패 재현 절차 + 릴리즈 게이트 (v1.0)

## 0) 문서 목적/범위
- 기준 여정: 촬영 → 라이브러리 선택/정리 → 브이로그 생성/처리 → 편집/AI 효과 → 내보내기 → 프로필/과금
- 플랫폼: **Android 전용**
- 우선순위 기준
  - **P0**: 릴리즈 차단급 (크래시, 데이터 유실, 핵심 플로우 불가)
  - **P1**: 핵심 기능 품질 저하 (우회 가능하나 사용자 영향 큼)
  - **P2**: 비핵심 이슈/UX 개선

## 2) 실패 케이스별 재현 절차 + 기대 결과

> 필수 포함 범주: **권한 / 저장공간 / 백그라운드 / 네트워크**

### FC-01 권한 거부: 카메라/마이크
- 우선순위: **P0**
- 재현 절차
  1. Android 설정에서 앱의 카메라/마이크 권한을 거부.
  2. 앱 실행 후 캡처 화면 진입.
  3. 녹화 버튼 탭.
- 기대 결과
  - 녹화 시작 금지, 크래시 없음.
  - 오류 메시지 예시: **"카메라/마이크 권한이 꺼져 있어요. [설정에서 권한 켜기] 또는 [다시 시도]를 눌러주세요."**
  - CTA: `설정으로 이동`, `다시 시도`

### FC-02 권한 거부: 저장소(사진/동영상 쓰기)
- 우선순위: **P0**
- 재현 절차
  1. 저장소/미디어 접근 권한 거부.
  2. 편집 완료 후 내보내기 실행.
- 기대 결과
  - 내보내기 즉시 실패 처리, 앱 무응답 없음.
  - 오류 메시지 예시: **"저장 권한이 필요해요. [설정으로 이동] 후 다시 내보내기 해주세요."**
  - CTA: `설정으로 이동`, `다시 내보내기`

### FC-03 저장공간 부족: 촬영 저장 실패
- 우선순위: **P0**
- 재현 절차
  1. 테스트 기기 저장공간을 200MB 미만으로 확보.
  2. 캡처 화면에서 3초 녹화 수행.
- 기대 결과
  - 저장 실패를 즉시 감지, 파일 손상/유령 항목 생성 없음.
  - 오류 메시지 예시: **"저장 공간이 부족해요. [공간 정리 후 다시 시도]를 눌러주세요."**
  - CTA: `저장공간 관리 열기`, `다시 시도`

### FC-04 저장공간 부족: 내보내기 실패
- 우선순위: **P0**
- 재현 절차
  1. 대용량 프로젝트를 연다.
  2. 저장공간 부족 상태에서 내보내기.
- 기대 결과
  - 실패율 집계에 반영(Export Failure).
  - 오류 메시지 예시: **"내보내기 공간이 부족해요. 최소 1GB 확보 후 [다시 내보내기] 해주세요."**
  - CTA: `공간 확보 가이드`, `다시 내보내기`

### FC-05 백그라운드 전환: 브이로그 처리 중 앱 스와이프 아웃
- 우선순위: **P0**
- 재현 절차
  1. 브이로그 생성 시작.
  2. 앱을 백그라운드로 보낸 뒤 최근 앱 목록에서 종료.
  3. 앱 재실행.
- 기대 결과
  - 작업 상태(취소/실패/재개 가능)가 일관되게 복원.
  - 오류 메시지 예시: **"앱이 종료되어 작업이 중단되었어요. [이어서 처리] 또는 [다시 시작]을 선택해주세요."**
  - CTA: `이어서 처리`, `다시 시작`

### FC-06 백그라운드 전환: 내보내기 중 화면 잠금
- 우선순위: **P0**
- 재현 절차
  1. 내보내기 시작.
  2. 화면 잠금 후 1~3분 대기.
  3. 잠금 해제 후 상태 확인.
- 기대 결과
  - 정책에 따라 백그라운드 지속 또는 일시중단 후 복구.
  - 손상 파일 생성 금지.
  - 안내 메시지 예시: **"백그라운드에서 내보내기를 계속하려면 [절전 해제]를 확인해주세요."**
  - CTA: `절전 설정 보기`, `계속`

### FC-07 네트워크 오프라인: 클라우드 동기화/업로드 실패
- 우선순위: **P1**
- 재현 절차
  1. 비행기 모드 ON.
  2. 클라우드 백업 또는 업로드 트리거.
- 기대 결과
  - 로컬 데이터 보존.
  - 오류 메시지 예시: **"인터넷 연결이 없어요. [오프라인으로 저장] 후 [연결되면 다시 업로드] 하세요."**
  - CTA: `오프라인 저장`, `재시도`

### FC-08 네트워크 불안정: 타임아웃 반복
- 우선순위: **P1**
- 재현 절차
  1. 네트워크 속도 제한(예: 2G/높은 지연) 환경 설정.
  2. 상품 조회/업로드/원격 리소스 처리 단계 실행.
- 기대 결과
  - 지정 횟수 재시도 후 사용자 선택지 제공.
  - 오류 메시지 예시: **"연결이 불안정해요. [지금 다시 시도]하거나 [나중에 시도]할 수 있어요."**
  - CTA: `다시 시도`, `나중에 하기`

## 3) 에러 메시지 정책 (행동 없는 메시지 금지)

### 3-1. 금지 규칙
- "오류가 발생했습니다"처럼 **행동 유도 없는 메시지 금지**
- 원인/행동/대안 중 최소 2개 이상 포함

### 3-2. 필수 템플릿
- 구조: **무엇이 문제인지 + 사용자가 지금 할 행동 + 대체 경로**
- 버튼은 1~2개 권장 (`다시 시도`, `설정으로 이동`, `오프라인 저장`, `나중에`) 

### 3-3. 문구 예시
- 권한: "권한이 꺼져 있어요. 설정에서 권한을 켠 뒤 다시 시도해주세요."
- 저장공간: "저장 공간이 부족해요. 공간을 정리한 뒤 다시 내보내기 해주세요."
- 네트워크: "인터넷이 불안정해요. 다시 시도하거나 나중에 업로드할 수 있어요."

## 4) 릴리즈 게이트 제안 (Android)

## 4-1. 차단 기준 (Blocking)

| 지표 | 차단 임계치 | 측정 구간 | 우선순위 |
|---|---|---|---|
| 세션 크래시율 (Crash-free Sessions 역산) | **> 0.30%** 이면 차단 (Crash-free < 99.70%) | 최근 7일, Android 전체 + 상위 5개 기기 | P0 |
| 내보내기 실패율 | **> 2.0%** 이면 차단 (사용자 취소 제외) | 최근 7일, 내보내기 500건 이상 표본 | P0 |
| 브이로그 생성 실패율 | **> 1.5%** 이면 차단 | 최근 7일, 생성 500건 이상 | P0 |
| 권한 관련 치명 실패율 | **> 0.5%** 이면 차단 (복구 불가 상태) | 최근 7일 | P0 |
| ANR 비율 | **> 0.20%** 이면 차단 | Play Vitals 기준, 최근 릴리즈 트랙 | P0 |

## 4-2. 경고 기준 (Warn, 릴리즈 가능/모니터링 강화)

| 지표 | 경고 임계치 | 조치 |
|---|---|---|
| 세션 크래시율 | 0.15% ~ 0.30% | 점진 배포 10% 유지 + 24시간 관찰 |
| 내보내기 실패율 | 1.0% ~ 2.0% | 실패 로그 샘플링/기기군 핫픽스 후보 추출 |
| 브이로그 생성 실패율 | 0.8% ~ 1.5% | 네트워크/코덱/저장공간 원인 분리 대시보드 점검 |

## 4-3. 릴리즈 의사결정 규칙
- 아래 중 1개라도 충족하면 **출시 차단**
  - P0 신규 버그 Open 상태
  - 4-1 표의 임계치 초과
  - 권한/저장공간/백그라운드/네트워크 시나리오 중 P0 테스트 미통과
- 모든 P0 테스트 통과 + 차단 지표 정상일 때만 프로덕션 승격
