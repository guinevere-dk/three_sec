## 2) 첨부 이미지 기반 UI 패턴 정리 (핵심 인사이트)

첨부 이미지에서 반복되는 공통 패턴:

1. **프리뷰 우선 레이아웃**
   - 상단: Back / 복원 / 저장
   - 중앙: 큰 프리뷰
   - 프리뷰 하단: 작은 반투명 재생바(모드에 따라 축소/대체)

2. **하단 2층 구조**
   - 1층: 모드별 서브패널(Angle/밝기/사운드 등)
   - 2층: 글로벌 툴바 아이콘(Trim, Transform, 밝기, AI, 사운드)

3. **선택값의 즉시 피드백**
   - 아이콘 선택 시 활성 하이라이트
   - 슬라이더 이동 중 아이콘/라벨이 숫자값으로 전환
   - 중앙 0점/기준점 강조(노란 기준선)

4. **컨텍스트 전환 시 타임라인 숨김**
   - Transform Angle, 밝기, 사운드 진입 시 클립 리스트를 숨기고 같은 자리에 서브패널 노출

5. **속도는 별도 확정형 프리셋**
   - Trim 상태에서 재생바 옆 속도 버튼 진입
   - 진입 후 재생바 영역을 0.25/0.5/1.0/2.0 고정형 버튼으로 대체

## 3) 요구사항 상세화 (기능별 UI/상태 정의)

## 3-1. 하단 메인 툴바 IA

### 툴바 순서 (고정)
`Trim | Transform | 밝기 | 사운드 | AI`

### 노출 규칙
- `Text`, `Sticker`는 메인 바에서 숨김 (기능 자체 제거 아님)
- 활성 탭은 아이콘+텍스트 색 강조
- 비활성 탭은 60~70% 불투명도

### 접근성
- 터치 타깃 최소 44dp
- 선택 상태는 색상 + shape(배경 원형/라운드) 2중 피드백

---

## 3-2. Trim 모드

### 기본 상태
- 하단 타임라인(클립 리스트) 노출 - 메모) 클립을 가로 확장해서 트림하는 기존 구조를 유지해라.
- 프리뷰 하단 재생바 노출

### Speed 진입 상태
- 재생바 가로폭 축소 + 오른쪽 `속도` 버튼 노출
- `속도` 버튼 탭 시:
  1) 재생바 + 속도버튼 숨김
  2) 동일 위치에 고정형 프리셋 버튼 `0.25 / 0.5 / 1.0 / 2.0`
  3) 선택값은 즉시 적용 및 하이라이트

### Speed 이탈
- 빈 영역 탭/뒤로/Trim 탭 재탭 시 기본 Trim 상태 복귀

### 값 정책
- 기본값 `1.0x`
- 표시 문구: `0.25x, 0.5x, 1.0x, 2.0x`

---

## 3-3. Transform 모드

### 기본 상태
- 프리뷰 위(재생바 상단)에 퀵버튼: `Flip / Rotate / Angle`
- 클립 리스트는 기본 노출(Angle 진입 전)

### 동작 정의
- **Flip**: 좌우 반전 토글
- **Rotate**: 탭할 때마다 +90° 누적 (0→90→180→270→0)
- **Angle**: Angle 서브모드 진입

### Angle 서브모드
- 클립 리스트 숨김
- 같은 자리에 카드형 패널 노출
- 상단 선택칩: `기울기 / 수평 / 수직`
- 하단 눈금자(룰러) + 중앙 기준선

### Angle 슬라이딩 UX
- 드래그 중:
  - 현재 선택된 항목 아이콘/라벨이 수치 텍스트로 전환
  - 0.1° 단위 업데이트
- 드래그 종료:
  - 값 커밋

### Direct Manipulation
- Transform 활성 시 프리뷰에서 pinch(확대/축소), pan(이동) 허용
- 목표: 프레임 드랍/깜빡임/강제 리빌드 체감 최소화

---

## 3-4. 밝기 모드

### 패널 진입
- 클립 리스트 숨김
- Angle과 동일한 패널 골격(높이, 카드 스타일, 눈금자 구조) 사용

### 속성 탭(가로 스크롤)
`밝기, 노출, 대비, 하이라이트, 그림자, 채도, 틴트, 색온도, 선명도, 명료도`

### 조작
- 선택된 속성 기준으로 룰러/슬라이더 조정
- 슬라이딩 중 아이콘/라벨 → 숫자값 전환
- 값 범위(권장):
  - 기본 톤 계열: `-100 ~ +100`
  - 색온도: `-100 ~ +100` (UI 텍스트는 Cool/Warm 보조 표기 가능)

### 초기화
- 속성별 개별 리셋 + 전체 리셋(패널 우상단)

---

## 3-5. 사운드 모드

### 패널 진입
- 클립 리스트 숨김
- 2개 슬라이더 노출:
  1) 전체 사운드(영상 원본)
  2) 클립 사운드(선택 클립/트랙)

### 표시 규칙
- 좌측 아이콘(스피커/클립)
- 중앙 슬라이더
- 우측 퍼센트 숫자(예: 75%)

### 권장 범위
- `0 ~ 100%`
- 기본값: 전체 100%, 클립 100%(또는 현재 시스템 값 유지)

## 3-6. AI 모드

이번 문서의 AI는 “생성형 기능 구현”이 아니라 **하단 UI 정렬/진입 UX** 중심이다.

- 하단 탭 명칭: `AI`
- 탭 진입 시 기존 Effects/AI 패널 진입점을 유지
- 시각 규칙:
  - AI 아이콘 강조색(노랑/퍼플 계열 중 1개 일관 선택)
  - 필요 시 PRO 배지 유지

- 슬라이더 드래그 중 모드 전환 시: `onChangeEnd` 강제 커밋 후 전환

## 5) 성능/UX 품질 기준 (중요)

요청사항의 핵심인 “끊김 없는 Transform UX”를 위해 아래 기준을 적용한다.

1. **실시간 제스처 구간 최소 빌드**
   - 제스처 중에는 프리뷰 트랜스폼 관련 값만 갱신
   - 하단 패널/타임라인 전체 리빌드 방지

2. **프레임 스케줄링 일관화**
   - 연속 입력은 프레임 단위 배치 반영
   - 드래그 종료 시 1회 커밋

3. **커밋 단위 통일 (Undo/Redo 친화)**
   - 버튼 탭: 1액션 1커밋
   - 슬라이더 드래그: 1드래그 세션 1커밋

4. **시각 안정성**
   - 패널 전환 애니메이션 180~220ms
   - 불필요한 스낵바/토스트 난발 금지

## 8) 수용 기준(AC)

1. 하단 메인 버튼이 `Trim/Transform/밝기/사운드/AI`로만 보인다.
2. Trim에서 Speed 진입 시 재생바가 사라지고 4개 고정 속도 버튼이 동일 위치에 노출된다.
3. Transform에서 Flip/Rotate/Angle 동작이 명확하고, Angle은 0.1° 단위 조절이 가능하다.
4. 밝기 패널은 Angle과 동일한 높이/룰러 구조를 사용한다.
5. 사운드 패널에서 전체/클립 2개 슬라이더를 독립 조절할 수 있다.
6. Transform 제스처 중 눈에 띄는 깜빡임/새로고침 체감이 없어야 한다.

## 10) Phase1 1차 테스트 피드백 반영 계획 (빠른 보완: Phase1.5)

Phase2로 넘어가기 전에, 아래 5개 이슈를 **짧은 수정 사이클로 먼저 정리**한다.

### 10-1. 이슈 목록 → 원인 가설 → 수정 방향

1. **Trim 진입 시 재생바 길이 축소 미반영 + 불필요 배경 도형 존재**
   - 원인 가설: Trim 오버레이를 단일 캡슐 컨테이너로 감싸면서 디자인 의도보다 큰 배경 레이어가 고정됨.
   - 수정 방향:
     - 재생바 영역과 속도 버튼 영역을 분리한다.
     - 배경은 재생바 하부에만 최소 반투명으로 적용하고, 속도 버튼 주변 배경은 제거/축소.

2. **재생바/속도 버튼이 화면 중앙으로 배치됨 (의도: 프리뷰 하단 고정)**
   - 원인 가설: `Align + Padding` 기준이 하단 프리뷰가 아닌 전체 Stack 기준으로 계산됨.
   - 수정 방향:
     - 프리뷰 하단 anchor 기준으로 절대 배치 규칙 고정.
     - 세로 위치 토큰(예: `previewBottomInset`)을 단일값으로 관리해 흔들림 제거.

3. **속도 버튼을 누르면 재생바 상단에 속도 프리셋이 떠야 함**
   - 원인 가설: 현재는 재생바 자체를 대체하는 구조여서 상/하 레이어 관계가 반영되지 않음.
   - 수정 방향:
     - `TrimMain`: 하단 재생바 + 우측 속도 버튼
     - `TrimSpeedPreset`: 기존 재생바는 유지하고, 프리셋 패널을 재생바 상단에 오버레이
     - 전환 애니메이션 180ms 이내(페이드+슬라이드)로 통일.

4. **속도 프리셋 진입 후 뒤로가기 동작 불량**
   - 원인 가설: 시스템 back 이벤트가 Trim 하위 상태를 우선 소모하지 않고 상위 라우트 pop로 전달됨.
   - 수정 방향:
     - back 우선순위 규칙 추가:
       1) `TrimSpeedPreset`이면 `TrimMain`으로만 복귀(화면 이탈 금지)
       2) `TrimMain`이면 Trim 종료
       3) 그 외 기본 back 동작
     - Android 물리 백버튼/제스처 백 모두 동일 규칙 적용.

5. **Trim 구역의 흰색 playhead를 드래그해 재생 위치 이동 불가**
   - 원인 가설: playhead가 표시 전용이며 hit-test/gesture가 핸들러에 연결되지 않음.
   - 수정 방향:
     - playhead 드래그 제스처 추가(`onHorizontalDragUpdate` 기반).
     - 드래그 중 debounce seek + 드래그 종료 시 1회 커밋(Undo 단위 유지).
     - 핸들 터치 타깃 최소 24~28dp로 확대.

5. 기존 클립 가로 확장 트림 UX(핸들 드래그)는 그대로 유지된다. = 성공.

## 12) Phase1 종결 실행계획 (마지막 정리, 구현 전)

11번 결과 기준으로 남은 쟁점은 **속도 프리셋 overflow/외곽 도형**과 **Trim playhead 드래그 seek 불가** 2건이다.  
12번에서는 이 2건을 닫고, **Phase2 진입 여부를 즉시 판정**하는 데 필요한 실행 순서/게이트를 고정한다.

### 12-1. 남은 실패 2건 재정의 (원인 확정)

1. **속도 프리셋 overflow + 외곽 배경 도형 노출**
   - 관측 신호: 로그에 `RenderFlex overflowed by 30 pixels on the right` 반복.
   - 확정 원인:
     - 프리셋 행의 라벨 길이(`0.25x / 0.5x / 1.0x / 2.0x`)와 버튼 최소 폭 합이 현재 가로 슬롯을 초과.
     - 프리셋 그룹 외곽을 감싸는 추가 데코 컨테이너가 있어 시각적으로 “배경 도형이 이중”으로 보임.
   - 종결 목표:
     - overflow 0건.
     - 외곽 추가 도형 제거(버튼 그룹 자체만 노출).

2. **Trim에서 playhead 드래그 seek 미동작**
   - 관측 신호: 재생 시 자동 이동은 되지만, 사용자 드래그 입력이 playhead seek으로 소비되지 않음.
   - 확정 원인:
     - playhead hit 영역보다 상위 레이어(타임라인/핸들/제스처 탐지기)가 포인터를 선점.
     - 드래그 시작점의 로컬 좌표/제약 구간(`startMs~endMs`) 반영 순서가 불안정해 업데이트가 무효화.
   - 종결 목표:
     - 탭 seek + 드래그 seek 모두 동작.
     - trim 핸들 조작 UX 품질은 유지.

### 12-2. 종결 구현 지시서(작업자용 체크리스트)

아래는 **구현 시 반드시 지킬 고정안**이다.

1. **속도 프리셋 행 컴팩트 규격화**
   - 라벨을 `1/4, 1/2, 1, 2`로 축약.
   - 버튼 가로폭/패딩을 compact 토큰으로 고정(균등폭 또는 최소폭 + `Expanded` 중 1안으로 통일).
   - 프리셋 그룹 바깥 장식 컨테이너(외곽 배경 도형) 제거.
   - 줄바꿈/overflow 발생 시 자동 축소 대신 레이아웃 폭 재분배를 우선.

2. **Trim 프리뷰 하단 바 구성 단순화(유지 정책 재확인)**
   - Trim 활성 중 프리뷰에는 `재생 버튼 + 시간표시 + 속도 버튼`만 표시.
   - 프리뷰 진행 슬라이더는 미노출(타임라인 playhead와 역할 중복 제거).

3. **playhead 입력 파이프라인 재연결**
   - playhead 시각선(얇은 선)과 별도 hit 영역(최소 24~28dp) 분리.
   - 제스처 우선순위:
     1) playhead drag/tap seek
     2) trim 핸들 drag
     3) 기타 스크롤/탭
   - seek 좌표는 타임라인 로컬 좌표만 사용(글로벌 좌표 금지).
   - 드래그 중 preview seek는 throttle, 드래그 종료 시 최종 1회 commit.

4. **회귀 방지 가드**
   - 속도 프리셋 열고/닫기 동작(성공 항목) 회귀 금지.
   - 백버튼 규칙(1회 프리셋 닫힘, 2회 Trim 종료, 화면 유지) 회귀 금지.
   - 기존 trim 핸들 가로 확장 조작감 유지.

2. Trim을 선택했을 때 하단 버튼이 X - Tiem Clip - V 으로 변하는데, 다른 버튼들과 UI가 매칭되지 않는다. 이 하단 버튼 X, Y는 없애고, 다른 버튼들처럼 활성화 색깔만 준다음 다른 탭으로 이동하거나 뒤로가기 하면 현상태를 저장하자.
3. Transform은 당초 원하던 내용이 하나도 반영이 안됐다. 3-3에서 정희한 내용 반영이 필요하다.
4. Transform 활성화 시, 퀵 바 아이콘 크기를 30%만 더 키우자.
5. 밝기에서 슬라이더와 눈금자가 중복이다. 나는 눈금자를 슬라이더처럼 좌우로 움직이면 해당 아이콘이 숫자로 바뀌면서 숫자가 보이길 원한다.
6. 밝기가 활성화된 상태에서 사운드를 선택하면 뒤에 묻혀버린다. 따로 검은 배경 도형을 뒤에 두지말고 Transform 처럼 반투명 배경에 조절을 할 수 있게 톤을 매칭해라.

## 14) 추가 개선 실행계획 (단계형, 구현 전)

13번 체크 결과와 하단 코멘트(6건)를 반영하여, **한 번에 전부 적용하지 않고 3단계로 분할**한다.

### 14-0. 원칙
- 이번 섹션은 **계획만 정의**하며 구현은 하지 않는다.
- 각 단계는 **진입 게이트(완료 기준)** 를 통과해야 다음 단계로 이동한다.
- 3-3/3-4 요구사항과 충돌 시, 본 섹션 우선이 아니라 **3번 요구사항 원문(상위 스펙)** 을 우선한다.

---

### 14-1. Step A — 하단 툴바/Trim 액션 정합성 복구 (저위험, 빠른 정리)

대상 이슈: 1번, 2번

1) **메인 툴바 텍스트 제거**
- 범위: `Trim / Transform / 밝기 / 사운드 / AI` 하단 버튼의 텍스트를 제거하고 아이콘 중심으로 정렬.
- 시각 규칙: 활성 상태는 색/배경으로만 구분(현재 활성 컬러 정책 유지).

2) **Trim 전용 X - Time Clip - V 바 제거/통합**
- 범위: Trim 진입 시 나타나는 하단 전용 바를 제거하고, 메인 툴바와 동일한 상호작용 모델로 통합.
- 저장 규칙:
  - 다른 탭 이동 시 현재 trim 상태 자동 커밋
  - 뒤로가기 시 trim 상태 자동 커밋 후 모드 이탈
- 회귀 방지: 기존 playhead/핸들 조작 품질 유지.

**Step A 완료 기준 (Gate-A)**
- [ ] 하단 툴바 텍스트가 전면 제거되고 아이콘만 노출됨
- [ ] Trim 전용 X/V 바가 사라지고 메인 툴바 모델로 동작
- [ ] 탭 전환/뒤로가기 시 trim 결과가 유실되지 않음

---

### 14-2. Step B — Transform 재구현(3-3 스펙 정렬)

대상 이슈: 3번, 4번

1) **3-3 요구사항 기준으로 Transform 재정렬**
- `Flip / Rotate / Angle` 퀵행동을 기준 축으로 재구성.
- Angle 진입 시 클립 리스트 숨김 + 카드형 패널 + `기울기/수평/수직` 칩 + 룰러 구조를 3-3 스펙대로 정렬.
- 드래그 중 숫자 전환/드래그 종료 1회 커밋 규칙 재검증.

2) **퀵 바 아이콘 30% 확대**
- 아이콘 자체 크기만 +30% 적용(패딩/터치타깃/행높이 균형 유지).
- 확대 후 overflow 발생 시 아이콘 간격/패딩 토큰을 재분배하되, 기능 축소는 금지.

**Step B 완료 기준 (Gate-B)**
- [ ] Transform 기능이 3-3 정의와 동일한 동작 흐름을 가짐
- [ ] 퀵 바 아이콘 30% 확대 적용 후 overflow 없음
- [ ] Angle 드래그 중 숫자 전환 + 0.1° 표기 유지

---

### 14-3. Step C — Brightness 입력 모델 단일화 + Sound 패널 레이어 정리

대상 이슈: 5번, 6번

1) **Brightness에서 슬라이더/눈금자 중복 제거**
- 입력 모델을 `눈금자(룰러) 중심 1개`로 단일화.
- 눈금자를 좌우로 드래그하면 값이 변하고, 드래그 중 해당 속성 라벨이 숫자로 전환되어 표시.
- 드래그 종료 시 1회 커밋(Undo/Redo 1세션 1커밋 유지).

2) **Brightness → Sound 전환 시 패널 톤/레이어 정합화**
- Sound 패널 배경을 Transform 계열 반투명 톤으로 맞춤.
- “뒤에 묻힘” 현상을 유발하는 별도 검은 배경 도형 제거.
- Brightness 활성 상태에서 Sound 진입 시 레이어 충돌 없이 즉시 전환.

**Step C 완료 기준 (Gate-C)**
- [ ] Brightness 입력 UI가 룰러 단일 모델로 동작
- [ ] 드래그 중 아이콘/라벨 → 숫자 전환이 안정적으로 노출
- [ ] Brightness 상태에서 Sound 진입 시 묻힘/겹침/블랙 배경 이슈 없음

---

### 14-4. 최종 검증 순서 (Step별 분할 테스트)

1. Step A 적용 후 즉시 회귀 테스트(Trim 저장/백동작/탭전환)
2. Step B 적용 후 Transform 전용 테스트(아이콘 확대, Angle UX, Direct Manipulation)
3. Step C 적용 후 Brightness/Sound 교차 테스트(레이어/톤/입력모델)
4. 모든 Step 완료 후 통합 게이트 재검증

---

### 14-5. 통합 게이트 (Step A~C 완료 판정)
- [ ] 하단 툴바 시각/동작이 단일 규칙으로 통합됨
- [ ] Transform이 3-3 스펙 기준으로 체감/동작 모두 일치
- [ ] Brightness와 Sound 전환 시 레이어 충돌 및 배경 이질감 없음
- [ ] Undo/Redo 커밋 단위(버튼 1회=1커밋, 드래그 1세션=1커밋) 유지
- [ ] Android 실기기 2종 이상에서 동일 동작 확인

### 검수 결과
1. 속도 버튼의 1/4~2 글자를 10% 더 작게 하고, 원형 버튼도 더 작게, 간격을 좁게 설정해라.
2. Tranform 활성화 시 나오는 퀵 바는 프리뷰 하단 재새방이 있던 위치로 고정해라.
3. Transform, 밝기, 사운드 활성화시의 프리뷰 크기는 Transform 기준으로 고정한다. 남은 공간에 맞춰서 기능들을 넣는다. 위 아래 간격을 더 줄여라
4. 밝기의 눈금자 아래 숫자를 추가로 표현해주는 항목은 제거해라. 아이콘에서 보여주는 숫자로 충분하다.
5. 아이콘에서 보여주는 숫자가 드래그할 때만 보여지고, 손을 때면 다시 글자로 돌아간다. 일단 슬라이드를 한 순간 계속 숫자로 보여주고, 옆에 다른걸 선택하면 그때 글자로 돌아오게 해라.
6. Trim에서 재생시켰을 때 Trim 구간안의 흰색 선이 재생 시간에 맞춰 움직여야하는데 안움직인다.

### 15-2-1. Step D-2b: 15-2 검수 4건 우선 해결(구현 전)

#### 대상
- 15-2의 하단 4개 검수 결과

#### 1) Quick 바 첫 클릭 반응성 복구
- 1차 클릭 즉시 반응 보장 우선순위
  - 현재 `Transform` 퀵 바 버튼 간 전환 경로를 **현재 활성 탭 갱신(비동기) + 상위 패널 열림/닫힘 분리**로 명세화.
  - 동일 프레임에서 state 토글/overlay 갱신이 겹치면서 반응이 지연되는 구간을 제거하고, `1회 포인터 업` 당 `1회 동작` 규칙으로 고정.
  - 동일 제스처 체인에서 `툴 전환` 이벤트와 `하위패널 상태 토글` 이벤트를 분기 처리해 2차 입력이 필요 없도록 상태전이 선행순서 고정.
- 정책:
  - 첫 클릭 즉시 목표 툴을 `pending`으로 표시.
  - 다음 프레임에서 패널 렌더가 완료되더라도 동일 탭 전환은 추가 클릭 없이 유지.

#### 2) Angle 아이콘 직관성 교체
- `Angle` 기능은 회전축 기울기 조정임을 즉시 인지할 수 있도록 아이콘을 교체.
- 후보 라인업: `선회 보정` 느낌의 `straighten`(또는 앱 아이콘셋 내 유사 아이콘)로 통일.
- 아이콘 변경은 라벨 텍스트(현재는 아이콘 중심 정책)와 충돌하지 않도록 15-1 규칙을 유지.

#### 3) Angle 인라인 패널 노출 위치 정합
- 15-2 Step D-2에서 정한 `공통 프리뷰 앵커`를 그대로 재사용.
- 현재 센터 오버레이처럼 보일 경우, 노출 위치를 **프리뷰 하단 클립리스트 레이어** 기준으로 고정.
- `Angle` 진입 시 패널은 동일 레이어에서 나타나되:
  - `Transform` 메인 퀵 바와 y축 기준 정렬 유지
  - 클립 리스트(타임라인) 구간과 겹치지 않는 단일 슬롯에서만 노출

#### 4) Angle 동작 모델 단일화(밝기와 동일 패턴)
- 입력 모델을 slider 2중 구조가 아닌 룰러 단일 모델로 통합.
- Angle 드래그 동작은 다음 순서를 따른다:
  1. 탭 시 선택 속성 즉시 표시 + 숫자 모드 전환 준비
  2. 드래그 중 해당 속성 라벨을 숫자 텍스트로 즉시 전환
  3. 드래그 종료 시 1회 커밋(Undo/Redo 세션 단위 유지)
  4. 다른 속성(또는 퀵 바 외부 영역 탭) 전환 시 라벨로 복귀

#### Step D-2b 완료 기준(Gate-D2b)
- [ ] Transform 퀵 바 버튼 첫 클릭/탭 변경이 1회 터치로 반응
- [ ] Angle 아이콘 교체 완료 및 변경으로 인한 클릭 실패/오탐 증가 없음
- [ ] Angle 패널이 프리뷰 하단 클립리스트 구역에서만 노출
- [ ] Angle 동작이 Brightness 동작 규약(룰러 단일, 드래그 숫자, 1회 드래그 커밋)과 일치

### 15-4: Step D-4: Trim 화이트 라인 동기화 복구

대상: 검수 6번
