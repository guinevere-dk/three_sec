# 로그인 직후 온보딩 튜토리얼 계획서 (초안)

## 1) 목표
- 새 사용자이거나, 기존 사용자라도 **앱 최초 설치 후 첫 로그인 시점**에 간단한 튜토리얼을 1회 노출한다.
- 튜토리얼은 핵심 사용 흐름 4가지를 빠르게 이해시키는 데 집중한다.

## 2) 튜토리얼 발동 조건

### 발동 대상
- 신규 계정 사용자 (회원가입 직후 첫 로그인)
- 기존 계정 사용자라도 현재 디바이스에서 앱 데이터가 초기 상태인 경우(재설치/앱 데이터 삭제 후 첫 로그인)

### 발동 시점
- 로그인 성공 직후, 메인 진입 전에 전체화면 튜토리얼(또는 모달 스텝)로 진입

### 재노출 정책
- 기본: 1회 노출
- 예외: 사용자가 튜토리얼을 끝까지 보지 못하고 중단한 경우, 다음 로그인에서 이어서 재개 또는 처음부터 재노출
- 설정(선택): 프로필 화면에서 “튜토리얼 다시 보기” 제공

## 3) 튜토리얼 구성 (4단계)

### Step 1. 촬영하기
- 메시지: “먼저 촬영 버튼을 눌러 클립을 기록해요.”
- 핵심 포인트:
  - 짧은 클립 단위 촬영
  - 촬영 후 라이브러리에 자동 저장됨

### Step 2. 클립 2개 이상 선택하기
- 메시지: “영상 제작을 위해 클립을 2개 이상 선택해 주세요.”
- 핵심 포인트:
  - 최소 선택 개수 조건(2개 이상)
  - 선택 상태 UI 강조(체크/카운트)

### Step 3. 영상 만들기
- 메시지: “선택한 클립으로 자동 편집 영상을 만들 수 있어요.”
- 핵심 포인트:
  - 생성 버튼 위치/동작 안내
  - 처리 중 상태(로딩/진행 안내) 표시
  - 결과물 확인 경로 안내

### Step 4. 외부 이미지/영상 가져오기
- 메시지: “갤러리에서 이미지나 영상을 가져와 함께 편집할 수 있어요.”
- 핵심 포인트:
  - 외부 미디어 가져오기 진입점 안내
  - 이미지/영상 모두 지원됨을 명시

## 4) 샘플 영상 파일명 제안 (2개)
튜토리얼 설명용 샘플로 아래 이름을 권장:

1. `tutorial_clip_capture_day.mp4`
2. `tutorial_clip_capture_night.mp4`

### 샘플 영상 저장 위치
- 권장 폴더: `assets/tutorial/videos/`
- 최종 경로 예시:
  - `assets/tutorial/videos/tutorial_clip_capture_day.mp4`
  - `assets/tutorial/videos/tutorial_clip_capture_night.mp4`

### pubspec 등록 규칙
- Flutter 번들 자산으로 사용하려면 `pubspec.yaml`의 `flutter > assets`에 아래 경로를 추가:
  - `assets/tutorial/videos/`

### 네이밍 가이드
- 소문자 + 언더스코어 사용
- 용도(`tutorial`), 유형(`clip`), 장면(`day/night`)이 이름에서 바로 드러나도록 구성
- 추후 다국어/서버 연동 시에도 충돌이 적은 영문 규칙 유지

## 5) UX 문구 예시 (짧은 버전)
- “촬영하고”
- “클립 2개 이상 고르고”
- “영상 만들고”
- “외부 사진/영상도 가져오세요”

## 6) 완료 기준 (Definition of Done)
- 로그인 직후 발동 조건이 정확히 동작한다.
- 4단계 튜토리얼 메시지가 실제 사용자 흐름과 일치한다.
- 샘플 영상 2개가 튜토리얼 화면에서 정상 재생된다.
- 튜토리얼 완료 상태가 저장되어 불필요한 재노출이 발생하지 않는다.
