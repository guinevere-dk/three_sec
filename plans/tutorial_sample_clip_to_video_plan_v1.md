# 튜토리얼 1~3 개선 계획서 (샘플 클립 기반 실제 영상 생성)

## 0. 문서 목적
- 현재 튜토리얼 1,2,3은 사용자 행동과 UI 포커스가 어긋나 있어 “무엇을 눌러야 실제 결과(영상 생성)가 나오는지” 전달력이 낮다.
- 본 계획은 **샘플 클립 2개를 앱 내부 `일상` 앨범에 노출**하고, 사용자가 이를 **선택 → 영상 만들기**까지 완료하도록 튜토리얼을 재구성하는 것을 목표로 한다.

---

## 1. 현 상태 진단 (코드/UX 기준)

### 1-1. 튜토리얼 단계와 실제 포커스 불일치
- 튜토리얼 문구 정의 위치: `MainNavigationScreen`의 튜토리얼 Phase 함수
  - `_showTutorialPhase1/2/3/4` in `lib/main.dart`
- 문제점:
  1) 2단계(클립 선택)와 3단계(영상 만들기)가 모두 사실상 라이브러리 상단/탭 포커스에 가까워, 사용자가 실제 액션 버튼을 인지하기 어렵다.
  2) “2개 이상 선택” 조건은 존재하지만, 선택 가능한 샘플 데이터가 첫 실행 시 항상 보장되지 않는다.
  3) 사용자 관점에서 “튜토리얼을 따라 했더니 정말 결과 영상이 만들어졌다”는 성공 경험이 약하다.

### 1-2. 실제 생성 동작 연결 상태
- 영상 만들기 실행은 `LibraryScreen`에서 다중 선택 후 액션패널의 `onCreateProject`를 통해 발생하고,
  최종적으로 `MainNavigationScreen._handleMerge()`로 전달됨.
- 병합/편집 진입은 이미 구현되어 있음:
  - 선택 경로 전달: `lib/screens/library_screen.dart`
  - 생성 처리: `lib/main.dart`의 `_handleMerge(List<String> selectedPaths)`

### 1-3. 샘플 에셋 상태
- 샘플 파일 존재:
  - `assets/tutorial/videos/tutorial_clip_capture_day.mp4`
  - `assets/tutorial/videos/tutorial_clip_capture_night.mp4`
- `pubspec.yaml`에 `assets/tutorial/videos/` 등록되어 있어 번들 로딩 가능.

## 2. 목표 UX (개선 후)

### 핵심 사용자 시나리오
1. 앱 첫 진입 시 튜토리얼 시작.
2. “샘플 클립 준비” 단계에서 샘플 2개가 `일상` 앨범에 자동 배치됨(중복 방지).
3. 사용자는 라이브러리에서 샘플 클립 2개를 선택.
4. “영상 만들기” CTA를 눌러 실제 병합/편집(또는 Free 빠른 내보내기) 실행.
5. 튜토리얼 완료 시점은 **실제 생성 흐름 진입 성공** 또는 **결과 생성 성공**을 기준으로 확정.

### 튜토리얼 문구 고정 정책 (최종 확정)
- 튜토리얼 코멘트는 `lib/main.dart`에 반영된 현재 문구를 **최종본**으로 간주한다.
- 본 계획서는 이후 문구 개선/톤 조정 대상에서 튜토리얼 코멘트를 제외한다.
- 즉, 본 문서에서의 튜토리얼 관련 작업은 **동작/게이트/포커스/노출 조건 개선만 허용**하며,
  코멘트 텍스트 변경은 허용하지 않는다.

#### 고정 대상 문구 (변경 금지)
- Step 1: "튜토리얼을 시작합니다." / "일상 앨범에 들어가보세요.\n영상을 만들 샘플 클립 2개를 넣어놓았습니다."
- Step 2: "클립 2개 이상 선택" / "클립을 길게 눌러 2개 이상 선택하세요."
- Step 3: "영상 만들기" / "클립을 2개 이상 선택 후 \n하단 \"영상 만들기\" 버튼을 누르면\n나만의 영상 앨범을 만들 수 있어요."
- Step 4: "외부 사진/영상 가져오기" / "우측 상단의 + 아이콘으로 외부 사진/영상을 불러와\n클립으로 변환할 수 있어요.\n"
- 완료: "튜토리얼 완료!" / "3s로 나만의 영상앨범을 간직해보세요!"

## 3. 구현 범위 (Phase 기준)

## Phase A. 샘플 클립 준비 레이어

### A-1. 샘플 클립 주입 유틸 추가
- 위치 제안: `lib/managers/video_manager.dart` 또는 튜토리얼 전용 유틸 파일
- 책임:
  1) 앱 문서 디렉터리 `raw_clips/일상`에 샘플 파일 복사
  2) 이미 존재하면 재복사하지 않음(파일명/마커 기반 idempotent)
  3) 소유권/인덱스/duration 캐시를 기존 흐름과 호환되게 반영

### A-2. 함수 시그니처 제안
- `ensureTutorialSampleClips({required String targetAlbum})`
- 반환값:
  - 실제 사용 가능한 샘플 경로 리스트 (2개)
  - 신규 복사 여부 플래그(optional)

### A-3. 데이터 정책
- 기본 타겟 앨범: `일상`
- 파일명 규칙(충돌 방지):
  - `clip_tutorial_day.mp4`
  - `clip_tutorial_night.mp4`
- 재설치/재로그인 시에도 중복 생성 없이 재사용 가능해야 함.

---

## Phase B. 튜토리얼 단계 재설계 (1~3 중심)

### B-1. 단계 목적 재정의 (문구 변경 없이)
- Step 1 (준비):
  - 기존 “촬영 버튼 안내”에서 “샘플 클립 자동 준비 + Library 이동” 중심으로 전환
  - 사용자에게 즉시 실행 가능한 재료(클립 2개) 제공
- Step 2 (선택):
  - 첫 번째 샘플 클립 그리드와 선택 상태(UI 카운트)를 직접 가리킴
  - “2개 선택 시 다음 버튼 활성화”를 문구에 명시
- Step 3 (실행):
  - 하단 `영상 만들기` 버튼(`keyCreateProject`)을 정확히 포커싱
  - 탭 영역이 아닌 실제 액션 버튼 위주 안내

### B-2. 튜토리얼 전환 게이트
- 단순 시간 지연 기반 전환 축소.
- 권장 게이트:
  - Step 2 완료 조건: 선택된 클립 수 >= 2
  - Step 3 완료 조건: `onMerge` 진입 이벤트 발생

### B-3. Skip 정책
- Skip 시에도 튜토리얼 완료 플래그는 저장 가능하되,
  “샘플 클립 준비”는 유지하여 이후 사용자가 라이브러리에서 바로 체험 가능하게 함.

---

## Phase C. 실제 영상 생성 성공 체감 강화

### C-1. 결과 피드백 문구
- Free 사용자: 빠른 내보내기 결과 메시지 명확화
- Standard 이상: 편집 화면 진입 시 “튜토리얼 목표 달성” 토스트/배지 고려

### C-2. 진단 로그 키 표준화
- 기존 `[Main][Diag][MergeFlow]` 로그 유지 + 튜토리얼 컨텍스트 추가
  - 예: `tutorial=true`, `sampleSelectedCount=2`

## 4. 코드 반영 포인트 (예정)

1) 튜토리얼 진입/완료 로직
- 파일: `lib/main.dart`
- 대상: `_checkAndStartTutorial()`, `_showTutorialPhase1/2/3()`, `_finishTutorial()`

2) 샘플 클립 보장 로직
- 파일: `lib/managers/video_manager.dart` (또는 `lib/services/` 신규 파일)
- 대상: 샘플 에셋 복사 + 앨범 반영 유틸

3) 라이브러리 선택/CTA 하이라이트 정밀도
- 파일: `lib/screens/library_screen.dart`
- 대상: `keyFirstClip`, `keyCreateProject` 기준 포커스 타겟 재검토

## 5. QA 체크리스트 (완료 기준)

## 필수 시나리오
- [ ] 첫 실행 시 `일상` 앨범에 샘플 클립 2개 노출
- [ ] 튜토리얼 2단계에서 사용자 선택으로 선택 카운트 증가 확인
- [ ] 2개 이상 선택 시 `영상 만들기` 버튼 활성화
- [ ] 버튼 탭 시 실제 `_handleMerge()` 경로 진입
- [ ] Free/Standard 각각에서 최소 1회 결과 생성 흐름 확인

## 회귀 시나리오
- [ ] 튜토리얼 스킵 후에도 일반 라이브러리 기능(삭제/이동/복사) 정상
- [ ] 앱 재실행 시 샘플 클립 중복 생성 없음
- [ ] 기존 사용자(튜토리얼 완료 사용자)에게 불필요한 재노출 없음

## 7. 구현 순서 제안
1. 샘플 클립 주입 유틸 구현 및 단위 검증
2. 튜토리얼 1~3 타겟/게이트 재설계 (문구는 고정 유지)
3. MergeFlow 이벤트와 튜토리얼 완료 조건 연결
4. QA 체크리스트 수행 및 로그 검증
5. 문구 변경 없이 QA 로그/동작 안정화 마무리

## 8. 결론
- 본 계획의 핵심은 “안내 텍스트”가 아니라 **사용자가 실제로 결과 영상을 만들어보는 성공 경험**이다.
- 샘플 클립을 튜토리얼의 시작점으로 고정하고, 선택/생성 버튼까지 행동을 정확히 연결하면
  튜토리얼 1~3은 의미 전달이 아닌 **실행 가능한 온보딩**으로 전환된다.
